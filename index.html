Basic terminal use

If you're already familiar with the Linux terminal, feel free to skip this section. For everyone else, these simple commands are essential for developing on Linux (and Mac). For reference, $ indicates that the commands following it should be entered into your terminal (don't put the $ into the terminal).

Changing directories

When you first open your terminal, it'll default to your home directory/folder. The path of your home directory should be /home/<your linux username>, eg. /home/james. To change directory, use $ cd <path to desitination folder>. To use an absolute path from your file system root, start with /. This will have the same result no matter what directory you're currently in. Eg. 

$ cd /home/<your linux username> # Change to your home directory  
$ cd ~                           # This shortcut does the same thing
$ cd ~/Documents

To use a relative path from your current directory, leave out the beginning /. Relative paths are mainly used in conjunction with ls to navigate around the file system. Eg.

$ cd ~          # Start in your home directory
$ cd Documents  # ~/Documents
$ cd FuncProg   # ~/Documents/FuncProg

Listing files and directories

If you want to list stuff in your current directory, you can use ls. Eg. 

$ cd ~
$ ls
Desktop Documents Downloads hello.txt Pictures

Usually coloured names are directories, while files are coloured normally.

You can also list hidden files and directories (which start with '.' in Linux), using ls -a or ls --all

$ cd ~
$ ls -a
.config Desktop Documents Downloads hello.txt Pictures
.ssh .world-domination-plan.txt

Making directories

You may also want to make directories using the terminal to quickly organise your work. To do that you can use mkdir <path to new directory>. Eg. 

$ cd ~
$ mkdir Haskell
$ cd Haskell
$ mkdir FuncProg Practice # You can make multiple directories at once
$ ls
FuncProg Practice  

This sequence of commands will create the directories ~/Haskell/FuncProg and ~/Haskell/Practice

Terminal autocomplete

A very useful feature of the terminal is partially typing out a command or path, then pressing tab to autocomplete part or all of it. The catch is that it only works when the autocomplete is unambiguous. Eg.

$ ls
great.hs greg.hs
$ ghci        # Now when we press tab...
$ ghci gre    

The gre autocompletes because the only two valid options in this directory start with gre. Pressing tab again will do nothing because both 'a' and 'g' are equally valid as the next letters in the file name.

$ ghci greg      # Now let's press tab again...
$ ghci greg.hs

The only valid file that starts with greg is greg.txt so the terminal can autocomplete it. This same principle applies to most things you do in the terminal.

Hello, World!

Task 0.1

Open your text editor and create a file called Hello.hs in a directory of your choosing. It's a good idea to use a directory that's linked to cloud storage (like git or Dropbox), that way you won't lose your work and you can easily transfer it all across machines.

Copy the following code into the file and save it (always remember to save):

module Main where
 
main :: IO ()
main = putStrLn "Hello, World!"

The first line announces that we are creating a new module called Main. This tells the compiler that the entry point to the program can be found here.

After that, we declare a function called main. This is a special function that serves as the entry point to your program. When the file is executed, the system will start running the code that is in this function.

The function is first given a type definition, where we write main :: IO (). This says that the main function will interact with IO, which stands for Input/Output. Note that :: is used for annotating the types of functions and values.

Task 0.2

Using the terminal, navigate to the directory you chose, then compile and run the code with these commands:

$ ghc Hello.hs # Compiles the code and outputs it to a file called 'Hello'
$ ./Hello      # Runs 'Hello'. './' indicates it's in the current directory

It should print the line Hello, World! to the terminal. This is because what putStrLn :: String -> IO () does is take a String (in this case "Hello, World!"), and sends that to the standard output (stdout), which by default prints to the terminal.

The type String -> IO (), means that you give the function a String and it gives you back something of type IO (), with IO indicating that it performs Input/Output and () (pronounced 'unit') indicating that it returns nothing useful.

Multiple Lines

What if you want to do multiple things? You can use do notation to chain together IO actions like so:

main :: IO ()
main = do
    putStrLn "Hello!"
    putStrLn "Bye!"

That's one way of using do notation, but the following is also valid:

main :: IO ()
main = do putStrLn "Hello!"
          putStrLn "Bye!"

It's mostly a matter of personal preference which one you choose. As long as all the code is aligned correctly it will compile. Just make sure to be consistent. And use spaces instead of tabs for indentation. Just do it.

Input

You can get input from the terminal using the getLine :: IO String function. Notice that this function performs Input/Output, but instead of returning () like main :: IO (), it returns a String instead. We can get that string by writing line <- getLine, which will bind a line of input from the standard input (stdin) into the variable line :: String. By default, requesting data from stdin will prompt the user to enter text into the terminal.

Task 0.4

Look at the following definition of main:

main :: IO ()
main = do
    line <- getLine
    putStrLn line

Try to figure out what this code is doing. Compile and run it to check if your understanding is correct.

Now edit it to reverse the user input before printing it back to the terminal.

Hint: reverse :: String -> String is defined in the Prelude.

Remember that putStrLn :: String -> IO () takes only one parameter, so you'll need to use round brackets/parentheses for it to type check. Try putting the parentheses in different places, then recompiling, to see where you get errors and where you don't, so you get a good understanding of what they're doing.

Notice that the type of reverse does not include IO, meaning it is a pure function. In other words, it has no side effects. In a functional language like Haskell, pure functions do what they say they do and nothing more. Functions that involve IO, like main, aren't pure and are therefore susceptible to the problem of side effects interacting in unexpected ways.

It can be tempting to spread IO all over the place to stop the compiler shouting at you, but the more functional your code the better. Your code will be easier to read, easier to debug, and won't turn into an unintelligible mess.

Show and Read

Two useful, pure functions for writing IO programs are show :: Int -> String and read :: String -> Int. These are simplified versions of the actual types, but we don't want to complicate things too much yet.

Show

show :: Int -> String takes in an Int and turns it into a String. Why would you want to do this? Well, let's look at this example:

main :: IO ()
main = putStrLn 5

If you try to compile this code the compiler will shout at you with a complicated error message, but here's the simplified issue: putStrLn :: String -> IO () takes a String as a parameter, but instead we've given it an Int.

So how do we solve this problem? We've got an Int (in this case 5) and we need it to be a String. That means we need a function that has the type Int -> String. Luckily we have one, it's called show! Let's use it:

main :: IO ()
main = putStrLn (show 5)

Now if you compile and run this code, it'll work exactly as you want it to.

This combination of putStrLn and show is quite useful, so they've been put together into their own little function:

print :: Int -> IO ()
print x = putStrLn (show x)

Which means print 5 is exactly the same as putStrLn (show 5).

Read

read :: String -> Int is the inverse of show. It's used when we want to take a String and turn it into an Int. When would we want to do that? Let's look at this example:

main :: IO ()
main = do
    number <- getLine
    print (negate number)

We take in a number :: String from the user by using getLine :: IO String, then we try to negate :: Int -> Int that number (ie. 5 becomes -5 and vice versa), and finally we print :: Int -> IO () the new number out. But there's a problem. number has type String, but we need it to be an Int instead, so that we can use negate :: Int -> Int.
